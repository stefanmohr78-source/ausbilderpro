⚠️ Firestore-Regeln prüfen
Damit Ausbilder, Lehrgangsarten und Berufsgruppen gespeichert werden können, muss shared/kursSettings in den Firestore Security Rules erlaubt sein.
Firebase Console → Firestore → Regeln — ersetze die bestehenden Regeln durch: 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isAdmin() {
      return request.auth != null
        && (get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'
         || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'verwaltung');
    }

    match /users/{uid} {
      allow read, write: if request.auth != null && request.auth.uid == uid;
      allow read: if isAdmin();
      match /{sub}/{subDoc} {
        allow read, write: if request.auth != null && request.auth.uid == uid;
        allow read: if isAdmin();
        match /{deep}/{deepDoc} {
          allow read, write: if request.auth != null && request.auth.uid == uid;
          allow read: if isAdmin();
        }
      }
    }

    match /shared/{docId} {
      allow read: if request.auth != null;
      allow write: if isAdmin();
      match /{sub}/{subDoc} {
        allow read: if request.auth != null;
        allow write: if isAdmin();
        // Ausbilder darf eigenen Urlaub aktualisieren
        allow update: if request.auth != null
          && request.resource.data.diff(resource.data).affectedKeys().hasOnly(['urlaub']);
        // Ausbilder/Admin darf Benachrichtigungen aktualisieren
        allow update: if request.auth != null
          && request.resource.data.diff(resource.data).affectedKeys().hasOnly(['benachrichtigungen']);
        // Urlaubslog: jeder darf Einträge erstellen
        allow create: if request.auth != null && docId == 'urlaubLog';
        match /{deep}/{deepDoc} {
          allow read: if request.auth != null;
          allow write: if isAdmin();
          // Ausbilder darf TN-Status + Attendance-Aggregat aktualisieren
          allow update: if request.auth != null
            && request.resource.data.diff(resource.data).affectedKeys().hasOnly(
              ['status','attTotal','attPresent','attAbsentE','attAbsentU','attSick','attBerufsschule','attLate','attEarly','attPercent','attLastSync']
            );
          // Azubi darf Kursbestätigung schreiben
          allow update: if request.auth != null
            && request.resource.data.diff(resource.data).affectedKeys().hasOnly(
              ['bestaetigtAm','bestaetigtVon']
            );
          // Ausbilder darf Dokumente hochladen/löschen (dokumente-Subcollection)
          allow create, delete: if request.auth != null && deep == 'dokumente';
        }
      }
    }

    match /globalTN/{doc} {
      allow read: if request.auth != null;
      allow write: if isAdmin();
      // Ausbilder darf nur status + wiederholer Felder ändern
      allow update: if request.auth != null
        && request.resource.data.diff(resource.data).affectedKeys().hasOnly(['status', 'wiederholer']);
      // Azubi darf eigenes Profilbild setzen
      allow update: if request.auth != null
        && request.resource.data.diff(resource.data).affectedKeys().hasOnly(['photoUrl']);
      // Azubi/Ausbilder darf Urlaub aktualisieren
      allow update: if request.auth != null
        && request.resource.data.diff(resource.data).affectedKeys().hasOnly(['urlaub']);
      // Azubi darf Benachrichtigungen als gelesen markieren + Kurs-Entfernung bestätigen
      allow update: if request.auth != null
        && request.resource.data.diff(resource.data).affectedKeys().hasOnly(
          ['benachrichtigungen','aktiverKursId','pendingRemoval','pendingRemovalKurs','kursHistory']
        );
      // TN-Dokumente (Krankmeldungen, Bescheinigungen)
      match /dokumente/{fileDoc} {
        allow read: if request.auth != null;
        allow create, delete: if request.auth != null;
      }
    }
  }
}

🚀 ERSTE SCHRITTE
1️⃣ Einstellungen – ⚙️ Berufsgruppen, Lehrgangsarten und Ausbilder anlegen.
2️⃣ TN anlegen – "+ Neuer Teilnehmer": Name, Berufsgruppe, Fachrichtung, Betrieb, Lehrjahr.
📥 TN importieren – CSV/XLSX Datei hochladen → Spalten zuordnen → Vorschau → Import. Optional: AzubiPro-Konten gleich miterstellen.
3️⃣ Kurs erstellen – "➕ Neuen Kurs erstellen": Ausbilder + Lehrgangsart + Zeitraum wählen, TN zuweisen.
📊 DASHBOARD
📊 Übersicht – Gesamt-TN, aktive Kurse, laufende Lehrgänge.
📋 Laufende Kurse – Liste mit Status, Ausbilder und TN-Anzahl.
👷 SIDEBAR – AUSBILDER
🏗️ Baumstruktur – Berufsgruppe → Ausbilder → Lehrgangsart → Lehrgänge.
⠿ Sortierung – Berufsgruppen per Drag & Drop oder A↓ alphabetisch sortieren.
👁 Ansicht – Augensymbol: Read-only Vorschau des Ausbilder-Sitzplans.
👥 SIDEBAR – TEILNEHMER
📂 Baumstruktur – Berufsgruppe → Lehrjahr → TN. Klick öffnet Profil.
📋 KURSDETAIL
📋 TN-Karten – Foto, Noten, Anwesenheits-%, Bestätigungsstatus.
✅ Bestätigung – "8/12 TN haben bestätigt". Pro TN: ✅ oder ⚠️.
📁 Kurs-Dokumente – Für alle sichtbar (Ausbilder + Azubis). Nur der Hochlader löscht.
📦 Archivieren – Beendete Kurse archivieren → TN werden freigegeben.
👤 TN-PROFIL
👤 Stammdaten – Name, Betrieb, Beruf, Lehrjahr, E-Mail, Telefon.
📷 Profilbild – Vom Azubi hochgeladen. Erscheint auf Sitzplankarte.
📅 Kurshistorie – Alle Lehrgänge. "+ Nachtragen" für vergangene Kurse.
📁 TN-Dokumente – Krankmeldungen, Bescheinigungen. Beide Seiten können hochladen.
⚙️ EINSTELLUNGEN (Admin)
🏢 Verwaltung-MA – Name + E-Mail → registriert sich selbst → automatischer Zugang.
👤 Ausbilder – Name + E-Mail + Telefon + Berufsgruppe. Telefon wird dem Azubi im Kursdetail angezeigt.
📋 Lehrgangsarten – z.B. "ET3", "ETE2". Dropdown beim Kurs-Erstellen.
🏗️ Berufsgruppen – Name + Emoji + Fachrichtungen (je Zeile eine).
📱 AzubiPro-Konto – Beim TN-Anlegen: Checkbox → Startpasswort → Zugangskarte drucken.
🏖 URLAUBSPLANUNG
🏖 Urlaubsübersicht – 🏖 Button in der Sidebar. Jahresübersicht aller Ausbilder nach Berufsgruppe.
👷 Ausbilder-Urlaub – Im Ausbilder-Profil unter Einstellungen eintragen.
👤 TN-Urlaub – Im TN-Profil eintragen. Azubis können ihren Urlaub auch selbst in AzubiPro eintragen.
🚫 Kurs-Builder – Ausbilder/TN mit Urlaub werden automatisch blockiert. Bei 1 Tag Urlaub entscheidet die Verwaltung.
🔔 Benachrichtigungen – Azubis werden automatisch benachrichtigt wenn ein Kurs gelöscht wird.
